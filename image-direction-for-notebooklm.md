# 포트폴리오 문서 이미지 배치 디렉션 (NotebookLM 전달용)

아래는 A4 포트폴리오 문서에 삽입할 이미지의 **정확한 위치, 종류, 크기, 목적**을 지정한 디렉션입니다.
이미지는 사용자가 직접 삽입하며, 이 가이드는 "어디에 무엇을 넣을지"를 상세히 안내합니다.

> 이미지 원본 소스: https://firebul.github.io/portfolio/assets/images/ 하위 경로

---

## 📄 1페이지 — 프로필 + 핵심 지표 + 역량 요약

### 이미지 ①: 프로필 사진
- **위치**: 페이지 우측 상단, 이름/연락처 옆
- **크기**: 가로 3cm × 세로 4cm (여권 사진 비율)
- **이미지 파일**: `profile.jpg`
- **설명**: 정장 착용 증명사진 스타일. 배경이 깔끔한 전문적 느낌의 상반신 사진
- **배치 방법**: 이름 "최원혁" + 연락처 블록의 우측에 정렬. 텍스트와 사진 사이 약 1cm 여백
- **스타일**: 모서리 약간 둥글게 (radius 4px 정도), 얇은 회색 테두리

### 이미지 ②: QR 코드
- **위치**: 프로필 사진 바로 아래 또는 1페이지 우측 하단
- **크기**: 2cm × 2cm
- **이미지 파일**: QR코드 생성기에서 아래 URL로 생성
  - URL: `https://firebul.github.io/portfolio-v2/`
- **목적**: 포트폴리오 사이트 바로 접속용. 아래에 "포트폴리오 사이트" 캡션 8pt

### 이미지 배치하지 않는 영역
- 핵심 지표 (4년 6개월, 50억원, +80%, 8개 모델, 5종 검출) → 텍스트/숫자 배지로 충분. 이미지 불필요
- Core Competence 3개 카드 → 아이콘이나 이미지 없이 텍스트만으로 구성

## 📄 2페이지 — 인터파크 주요 프로젝트 상세 (쿠폰 + 광고)

### 이미지 ③: 최저가 쿠폰 자동화 시스템 — 시스템 아키텍처 다이어그램
- **위치**: "최저가 쿠폰 자동화 시스템" 프로젝트 설명 바로 아래, 본문 전체 너비
- **크기**: 가로 전체 너비 (약 17cm) × 세로 5~6cm
- **이미지 파일**: `projects/coupon-flow.png`
- **설명**: 네이버/에누리/다나와 → Kafka → 데이터 처리 → 쿠폰 발행의 전체 흐름을 보여주는 시스템 아키텍처 플로우차트. 보라색 계열의 박스와 화살표로 구성된 다이어그램
- **목적**: "데이터 기반 의사결정"과 "기술적 깊이"를 시각적으로 증명
- **캡션**: "[그림 1] 최저가 쿠폰 자동화 시스템 아키텍처" (9pt, 회색)

### 이미지 ④: 상품 최종 지면 광고 개선 — A/B 테스트 Before/After
- **위치**: "상품 최종 지면 광고 개선" 프로젝트 설명 바로 아래
- **크기**: 가로 전체 너비, 2분할 (좌: AS-IS / 우: TO-BE), 각 8cm × 6cm
- **이미지 파일**: `projects/ads-result.png`
- **설명**: 좌측에 AS-IS(기존 상품 상세페이지 하단), 우측에 TO-BE(쎈딜 상단 배치 후) 비교 스크린샷. 실제 인터파크 상품 상세페이지의 "함께 보면 좋은 상품" 영역과 광고 배치가 변경된 모습
- **구성 방식**:
  - 좌측 이미지 위에 "AS-IS" 라벨 (빨간 점선 테두리)
  - 우측 이미지 위에 "TO-BE" 라벨 (초록 점선 테두리)
  - 가운데에 화살표(→) 아이콘
- **목적**: A/B 테스트의 구체적인 실험 설계를 시각적으로 보여줌 → PM의 과학적 접근법 증명
- **캡션**: "[그림 2] 상품 상세 광고 지면 A/B 테스트: 기존안 vs 쎈딜 상단 배치" (9pt)

### 선택 이미지 ⑤: A/B 테스트 결과 테이블 (있으면 좋음)
- **위치**: AS-IS/TO-BE 이미지 바로 아래
- **크기**: 가로 전체 너비 × 세로 3cm
- **내용**: 테이블 형태로 결과 요약
  | 지표 | A안 (기존) | B안 (쎈딜 상단) |
  |------|-----------|---------------|
  | 클릭률 | 0.36% | **0.70%** (↑94%) |
  | 거래 전환율 | 4.7% | **4.8%** |
- **목적**: 데이터 리터러시를 숫자로 직접 증명
- 이 테이블은 이미지가 아닌 텍스트 테이블로 넣어도 OK

## 🎨 전체 디자인 가이드라인

### 이미지 스타일 통일
- 모든 스크린샷/다이어그램: 얇은 회색 테두리 (1px, #E5E7EB)
- 모든 캡션: 9pt, 회색 (#6B7280), 이미지 바로 아래 중앙 정렬
- 이미지 모서리: 일관되게 약간 둥글게 (4px radius)

### 이미지 비중 권장
| 페이지 | 이미지 수 | 텍스트:이미지 비율 |
|--------|----------|-------------------|
| 1페이지 | 1~2개 (프로필+QR) | 85:15 |
| 2페이지 | 2~3개 (쿠폰 아키텍처+광고 AB) | 55:45 |
| 3페이지 | 1~2개 (CRM 푸시+광고센터) | 70:30 |
| 4페이지 | 2개 (AI 트레이딩 엔진 대시보드+Teflon) | 50:50 |
| 5페이지 | 1~2개 (앙상블 단체사진+공연) | 75:25 |

### 우선순위 (공간이 부족할 때 이 순서로 생략)
1. **필수**: ① 프로필 사진, ④ A/B 테스트 Before/After, ⑧ AI 트레이딩 엔진 대시보드
2. **강력 추천**: ③ 쿠폰 아키텍처, ⑥ CRM 푸시 목업, ⑩ 앙상블 단체사진
3. **있으면 좋음**: ② QR코드, ⑦ 광고주 센터, ⑨ Teflon, ⑪ 병원 공연

### 절대 하지 말 것
- 텍스트와 이미지가 겹치는 배치
- 이미지만 있고 캡션/설명 없는 배치
- 한 페이지에 3개 초과 이미지 배치 (답답해 보임)
- 이미지 크기가 페이지 너비의 50% 미만인 스크린샷 (너무 작아서 안 보임)
