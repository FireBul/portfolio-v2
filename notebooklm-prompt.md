### 프로젝트 1: 최저가 쿠폰 자동화 시스템 구축

**역할**: PM · 기획 · 데이터 분석
**기간**: 인터파크 SRM팀 재직 중 전 기간
**성과**: 월 50억원 거래액 / YoY 거래액 +30% / YoY 수익성 +20%

**배경 및 문제 정의:**
인터파크 쇼핑은 네이버 쇼핑, 에누리, 다나와 등 가격 비교 채널을 통해 유입되는 고객이 매우 많은 구조였습니다. 이 채널들에서 최저가 노출을 확보하는 것이 매출에 직결되었으나, 기존에는 상품별 쿠폰을 수동으로 관리하고 있어 경쟁사 대비 가격 대응 속도가 느리고, 쿠폰 마진율 관리가 비효율적이었습니다.

**수행 내용:**
- 네이버 EP(Electronic Publishing) API에서 전달되는 상품 데이터를 기반으로 실시간 가격 비교 자동화 쿠폰 시스템을 기획
- 기존 데이터 수집 방식의 한계를 파악하고, Kafka 기반 신규 topic을 설계하여 데이터 파이프라인을 전면 재구축
- 소카테고리별로 상품 등급을 세분화하여 등급별 쿠폰 마진율을 차등 적용하는 정책 수립
- 원부(원가 기준 부과) 방식의 최저가 쿠폰 운영 로직을 기획하고, 네이버·에누리·다나와 3개 채널에 동시 적용
- 이익률 하락 이슈 발생 시 소카테고리별 상품 등급 조정 → YoY 월평균 수익성 20% 개선
- 신규 제휴사 역 EP(제휴사가 인터파크에 보내는 데이터) 활용 자동화 쿠폰 확장

**사용 도구**: 네이버 EP API, Kafka, Python, MySQL, Tableau

**핵심 성과:**
| 지표 | 수치 |
|------|------|
| 월 거래액 | 50억원 |
| YoY 거래액 성장 | +30% |
| YoY 수익성 개선 | +20% |
| 적용 채널 | 네이버, 에누리, 다나와 |

### 프로젝트 2: 상품 최종 지면 광고 개선

**역할**: PM · A/B 테스트 설계 · 데이터 분석
**기간**: 인터파크 SRM팀 재직 중
**성과**: 광고액 +80% / CTR +100% / UX 품질 유지

**배경 및 문제 정의:**
인터파크 쇼핑의 상품 상세페이지는 구매 전환이 가장 많이 일어나는 핵심 지면이었으나, 해당 페이지에는 광고 영역이 없었습니다. 광고 영역을 신설하면 수익은 늘어나겠지만, 잘못 배치하면 사용자 경험을 해칠 수 있다는 우려가 있었습니다.

**수행 내용:**
- 상품 상세페이지 내 광고 영역 신설을 기획하고, 사용자 경험에 미치는 영향을 최소화하기 위해 A/B 테스트 방식으로 접근
- Google Analytics를 통한 사용자 행동 데이터 분석 → 광고 배치 위치 3곳 후보 도출
- A/B 테스트를 설계하여 각 위치별 광고 노출 효과와 사용자 이탈률을 비교 검증
- 테스트 결과, UX 품질(이탈률, 체류 시간)은 유지하면서 광고액 80%, CTR 100% 상승하는 최적 배치를 확정
- 결과를 토대로 상품 최종 지면 광고를 정식 도입

**사용 도구**: A/B Testing 프레임워크, Google Analytics, MySQL

**핵심 성과:**
| 지표 | 수치 |
|------|------|
| 광고액 상승 | +80% |
| CTR 상승 | +100% |
| UX 품질 | 유지 (이탈률/체류시간 동일) |

### 프로젝트 3: 바이럴 광고 CPS 모델 도입

**역할**: PM · 제휴사 관리 · 수익 모델 설계
**기간**: 인터파크 SRM팀 재직 중
**성과**: 4개 제휴사 CPS 수익 모델 구축 / 정산 프로세스 표준화

**배경 및 문제 정의:**
인터파크는 기존 CPC(Cost Per Click) 중심의 광고 수익 모델을 운영하고 있었으나, 바이럴 마케팅 채널에서는 클릭 기반 과금보다 실제 판매 성과 기반(CPS, Cost Per Sale) 과금이 더 효율적이었습니다. 그러나 CPS 모델을 운영하기 위한 시스템이 없었고, 제휴사별 정산 방식도 표준화되어 있지 않았습니다.

**수행 내용:**
- 11시11분, 커넥트온, 쿠차, 와이즈버즈 등 4개 바이럴 광고 제휴사와 CPS(판매 기반) 수익 모델을 설계
- 제휴사별 트래킹 코드 체계, 실적 집계 기준, 수수료 정산 로직을 표준화
- 실적 관리를 위한 백어드민 시스템을 기획하고 구축 → 제휴사별 매출, 전환율, 수수료를 실시간 관리
- Power BI를 활용한 제휴사 성과 대시보드를 구축하여 채널·지면별 성과를 체계적으로 시각화
- 정산 프로세스를 표준화하여 제휴사 정산 관련 분쟁 최소화

**사용 도구**: Power BI, Excel, 백어드민 시스템, MySQL

**핵심 성과:**
| 지표 | 내용 |
|------|------|
| 제휴사 수 | 4개사 (11시11분, 커넥트온, 쿠차, 와이즈버즈) |
| 수익 모델 | CPC → CPS 전환 |
| 시스템 | 백어드민 + 정산 프로세스 표준화 |
| 성과 관리 | Power BI 대시보드 구축 |

### 프로젝트 4: CRM 개인화 광고 운영

**역할**: PM · CRM 시나리오 설계 · 데이터 분석
**기간**: 인터파크 SRM팀 재직 중
**성과**: 개인화 CPC 광고 시나리오 체계 구축

**배경 및 문제 정의:**
인터파크의 기존 광고 운영은 전체 사용자에게 동일한 배너·푸시를 노출하는 일괄형 방식이었습니다. 사용자 개인의 관심사, 최근 검색/구매 이력에 맞춘 개인화 광고의 필요성이 대두되었으나, 이를 자동화하는 시나리오 체계가 없었습니다.

**수행 내용:**
- Braze(CRM 마케팅 자동화 툴)와 GA(Google Analytics)를 연동하여 사용자 행동 데이터 기반 개인화 타겟팅 구현
- 사용자 세그먼트별 맞춤 시나리오 설계: 장바구니 이탈 리마인드, 카테고리별 관심 상품 추천, 재방문 유도 등
- 개인화 푸시 알림 및 배너 광고 시나리오를 기획하고 A/B 테스트로 최적화
- CPC 시나리오별 성과 분석 → 전환율이 높은 시나리오 확대, 저성과 시나리오 개선/중단 판단

**사용 도구**: Braze, Google Analytics, CPC 시나리오 설계

### 프로젝트 5: 외부 채널 매체 확장

**역할**: PM · 채널 기획 · 연동 관리
**기간**: 인터파크 SRM팀 재직 중
**성과**: 신규 채널 2곳 연동 / 광고 수익원 다변화

**배경 및 문제 정의:**
인터파크의 CPC 광고 수익은 기존 내부 채널에 의존도가 높았습니다. 광고 수익원을 다변화하고 신규 트래픽을 확보하기 위해 외부 매체 채널 확장이 필요했습니다.

**수행 내용:**
- 넥스트페이퍼, 노티플러스 등 신규 외부 광고 채널 2곳을 발굴하고 연동 기획
- 각 채널의 노출 방식, 과금 모델, 타겟 사용자 특성을 분석하여 인터파크 상품과의 핏(fit) 검증
- 채널별 광고 연동 사양서 작성 및 개발팀과 협업하여 API 연동 진행
- 연동 이후 채널별 광고 성과(노출수, 클릭수, 전환율, ROAS)를 모니터링하고 광고액 배분 최적화
- 외부 채널 추가를 통해 광고 수익원이 다변화되어 특정 채널 의존 리스크 감소

**사용 도구**: 채널별 광고 API, Google Analytics, Power BI

### 프로젝트 6: 판매자 광고센터 ADMIN 운영

**역할**: PM · 광고 정책 수립 · 품질관리
**기간**: 인터파크 SRM팀 재직 중 전 기간
**성과**: RTB 과금 체계 구축 / 고위험 광고 품질관리 체계 확립

**배경 및 문제 정의:**
인터파크의 판매자 광고센터는 판매자가 직접 광고를 등록하고 관리하는 셀프서브(self-serve) 시스템입니다. 판매자 수가 증가하면서 고위험 광고(소프트웨어, 건강기능식품 등 규제 대상 상품)의 노출 리스크가 커졌고, RTB(Real-Time Bidding) 과금 정책과 채널별 광고액 CAP(상한) 관리도 필요해졌습니다.

**수행 내용:**
- 판매자 광고센터 ADMIN 게시판 전체 운영: 광고 등록 검수, 게시 승인/반려, 위반 광고 관리
- 고위험 광고(소프트웨어, 건강기능식품 등) 검수 기준을 수립하고 검수 프로세스 표준화
- RTB(실시간 입찰) 과금 정책 수립: 입찰 단가 산정 기준, 과금 방식, 부정 클릭 필터링 정책
- 채널별 광고액 CAP(상한) 정책 수립: 채널별 일일/월별 광고액 상한을 설정하여 과다 지출 방지
- 판매자 광고 관련 CS(고객 서비스) 처리: 광고 장애 대응, 판매자 문의 처리, 환불/보상 정책 운영
- 채널별/지면별 실적 관리: Power BI, Excel로 주간/월간 실적 리포트 생성

**사용 도구**: 판매자 광고센터 ADMIN, Power BI, Excel, MySQL

### 프로젝트 7: 데이터 수집 구조 개선

**역할**: PM · 데이터 파이프라인 기획
**기간**: 인터파크 SRM팀 재직 중
**성과**: YoY 월평균 거래액 +30%

**배경 및 문제 정의:**
기존 쿠폰 자동화 시스템의 데이터 수집 구조는 배치(batch) 방식으로 운영되고 있어, 가격 비교 채널의 실시간 가격 변동에 대응하는 속도가 느렸습니다. 특히 경쟁사가 가격을 내리면 인터파크의 쿠폰 갱신까지 시간 지연이 발생하여 최저가 노출 기회를 놓치는 경우가 빈번했습니다.

**수행 내용:**
- 기존 배치 기반 데이터 수집의 한계를 분석하고, 실시간 스트리밍 처리가 필요한 데이터 항목을 정의
- Kafka를 활용한 신규 topic 설계: 가격 변동 이벤트, 재고 변동 이벤트, EP 갱신 이벤트 등을 분리
- 개발팀과 협업하여 기존 수집 구조에서 Kafka 기반 파이프라인으로 전환하는 마이그레이션 기획
- 전환 후 가격 비교 채널 대응 속도가 대폭 개선되어 최저가 노출 빈도 증가
- 결과적으로 YoY 월평균 거래액 30% 증가 달성

**사용 도구**: Kafka, Python, MySQL

### 프로젝트 8: 쿠폰 정산 로직 개선

**역할**: PM · 정산 정책 기획
**기간**: 인터파크 SRM팀 재직 중
**성과**: 판매자·플랫폼 간 공정성 확보

**배경 및 문제 정의:**
기존 쿠폰 정산 방식은 할인 후 판매가액(실결제금액)을 기준으로 수수료를 부과하고 있었습니다. 그러나 쿠폰 할인이 커질수록 수수료 기준 금액이 낮아져 플랫폼 수익이 감소하는 구조적 문제가 있었고, 동시에 판매자 입장에서도 할인 부담에 대한 불만이 있었습니다.

**수행 내용:**
- 기존 정산 로직(할인 후 판매가 기준 수수료)의 구조적 문제를 데이터로 분석하고, 이해관계자(판매자, 재무팀, 운영팀)의 의견 수렴
- 할인 전 판매가액(원가) 기준으로 수수료를 부과하는 새로운 정산 로직을 기획
- 변경 시 판매자 부담 변화를 시뮬레이션하여 영향도 분석 리포트 작성
- 재무팀, 법무팀과 협업하여 정산 정책 변경에 따른 계약 조건 업데이트
- 판매자 커뮤니케이션: 정산 방식 변경 사전 공지 및 FAQ 작성
- 결과적으로 판매자·플랫폼 모두에게 공정한 정산 구조 확립

**사용 도구**: Excel, MySQL, 정산 시뮬레이션
